# Conclusiones
---

**Dificultades** 
- Definir una arquitectura clara y escalable par ala aplicación web de fitness.
- Integrar las distintas capas (Frontend, Backend y Base de datos) de forma coherente
- Elegir tecnologías adecuadas para cada capa de la aplicación ficticia.

**Aprendizajes** 
- Aplicar la arquitectura MVC en el proyecto web.
- Documentar procesos técnicos de instalación y uso de forma clara y profesional.
- Redactar documentación técnica con Markdown.

**Posibles mejoras** 
- Añadir soporte para autenticación social (Google, Facebook) para facilitar el registro de usuarios.
- Implementar una API REST para permitir integración con aplicaciones móviles.
- Mejorar la visualización de estadísticas y progresos mediante gráficos interactivos.
